# 2. Functional Requirements

Requirements follow [ISO/IEC 29148](https://www.iso.org/standard/72089.html). Each requirement is self-contained with a measurable acceptance criterion.

**Traceability:** These SRS IDs map to validated requirements in the project's internal `PROJECT.md`. All requirements listed here are verified against a running instance (v1.0.0 and v1.1.0 Hardware-UAT on Prusa Core One).

## Requirements

| ID      | Requirement                                                                                                                                                                      | Acceptance Criterion                                                                                                                                                             |
| ------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SRS-001 | The system shall read printer configuration (PrusaLink URL, credentials, Obico server URL, API key) from a JSON file on a Docker volume.                                         | `GET /api/config` returns the current configuration. After editing and saving, the bridge reloads the new config without container restart.                                      |
| SRS-002 | The system shall connect to the PrusaLink HTTP API using HTTP Digest Authentication.                                                                                             | A connection test to a live PrusaLink endpoint with valid credentials returns HTTP 200. Invalid credentials return a connection error reported to the user.                      |
| SRS-003 | The system shall poll the PrusaLink status endpoint (`/api/v1/status`) and job endpoint (`/api/v1/job`) at a configurable interval (default: 5 s).                               | Printer state changes (IDLE → PRINTING) are reflected in the Obico dashboard within 2× the configured interval.                                                                  |
| SRS-004 | The system shall relay pause, resume, and cancel commands received from Obico to the PrusaLink API.                                                                              | Sending a pause command from Obico causes the printer state to transition to PAUSED within 10 s. Resume and cancel behave equivalently.                                          |
| SRS-005 | The system shall capture the printer's RTSP camera stream via ffmpeg and extract JPEG frames at a configurable interval (default: 10 s).                                         | The camera module emits at least one JPEG frame within 30 s of starting when the RTSP endpoint is reachable.                                                                     |
| SRS-006 | The system shall upload JPEG camera frames to the Obico server via HTTP POST to `/api/v1/octo/pic/`.                                                                             | The Obico server displays an up-to-date camera image in the printer detail view within 2× the configured frame interval.                                                         |
| SRS-007 | The system shall maintain a persistent WebSocket connection to the Obico server using the Moonraker agent protocol (`moonraker_obico` v2.1.0).                                   | After an initial pairing, the bridge reconnects automatically within 60 s of a WebSocket disconnect without manual intervention.                                                 |
| SRS-008 | The system shall implement an Obico pairing flow: generate a 5-character pairing code and poll for server confirmation.                                                          | The setup wizard displays a pairing code. After the user links the printer in Obico, the wizard advances automatically within 120 s.                                             |
| SRS-009 | The system shall provide a browser-based setup wizard (4 steps: PrusaLink → Camera → Obico Pairing → Done) that guides a first-time user to a fully configured state.            | A user with no prior configuration can complete the wizard and reach a functioning bridge state in under 10 minutes.                                                             |
| SRS-010 | The system shall provide a live dashboard showing printer connection status, camera preview, and print controls (pause/resume/cancel).                                           | The dashboard updates printer state within 2× the polling interval. Camera preview shows a frame not older than 2× the frame interval.                                           |
| SRS-011 | The system shall provide an MJPEG snapshot stream endpoint (`/stream`) for compatibility with Obico's snapshot-based monitoring.                                                 | A browser or HTTP client can consume `GET /stream` and receive a continuous MJPEG stream while the camera is running.                                                            |
| SRS-012 | The system shall relay a WebRTC live stream via a bundled Janus gateway to the Obico control panel.                                                                              | The Obico control panel shows a live WebRTC stream from the printer camera with latency under 3 s on LAN.                                                                        |
| SRS-013 | The system shall apply circuit breaker protection to the PrusaLink, Obico WebSocket, and camera connections.                                                                     | After 5 consecutive failures on a connection, the circuit breaker opens. The dashboard health UI reflects the open state. The breaker resets after 60 s or on manual reconnect.  |
| SRS-014 | The system shall handle SIGTERM by closing the HTTP server, disconnecting the Obico WebSocket, and stopping the camera (ffmpeg) within 8 s.                                      | `docker stop` completes within 10 s. No orphaned ffmpeg processes remain after shutdown.                                                                                         |
| SRS-015 | The system shall list, upload, print, and delete G-code files on the connected PrusaLink storage.                                                                                | A file uploaded via the dashboard appears in the file list. Selecting "Print" starts the file on the printer. Delete removes it from both the list and PrusaLink.                |
| SRS-016 | The system shall expose OctoPrint-compatible file management endpoints (`GET /api/files`, `POST /api/files`, `DELETE /api/files/{path}`) to support Obico's remote file browser. | Obico's file browser tab shows the printer's file list. File upload via Obico completes for files up to 200 MB without an out-of-memory error.                                   |
| SRS-017 | The system shall accept print-start commands from Obico (`start_printer_local_print` and `file_downloader.download`) and relay them to PrusaLink.                                | A print initiated from the Obico file library or control panel starts on the printer within 10 s.                                                                                |
| SRS-018 | The system shall validate the origin of file download URLs received from Obico to prevent SSRF attacks.                                                                          | A `file_downloader.download` command with a URL from a host other than the configured Obico server is rejected with a logged error; no HTTP request is made to the foreign host. |
